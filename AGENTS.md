# Breakshot Repository Map

## Client
- `scenes/` contains Godot scenes.
- `scripts/Main.gd` is the current Godot client shell. It is grouped by feature order: connection/rooms, input, audio, state interpolation, drawing, layout helpers.
- Prefer adding small helper scripts under `scripts/features/<feature>/` for new isolated client logic instead of growing `Main.gd` further.

## Server
- `server/index.js` owns process wiring: HTTP/static serving, WebSocket messages, room lifecycle, simulation tick, snapshots.
- `server/src/rules.js` owns shared gameplay constants, default rules, and rule sanitization.
- New server features should go under `server/src/<feature>.js` and be imported by `server/index.js`.

## Assets
- `assets/game/` contains the gameplay texture atlas and atlas metadata.
- `assets/ui/` contains UI button atlases.
- `assets/backgrounds/` contains arena backgrounds.
- `assets/effects/` contains runtime VFX textures.
- `assets/legacy/` contains old/reference assets that are not part of the current runtime path.

## Generated Output
- `web_export/` is generated by Godot export but is committed because the Docker deployment serves it directly.
- Re-export after changing scripts or imported assets.
